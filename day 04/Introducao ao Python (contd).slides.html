Tuplas¶

🚨 Uma tupla é uma sequência imutável, de tamanho fixo, que armazena objetos do Python

Você pode criar uma tupla...
In [1]:
BRICS = ('Brasil', 'Russia', 'India', 'China', 'Africa do Sul')
BRICS
Out[1]:
('Brasil', 'Russia', 'India', 'China', 'Africa do Sul')
... você pode acessar uma tupla...
In [2]:
BRICS[0]
Out[2]:
'Brasil'
In [3]:
BRICS[1]
Out[3]:
'Russia'
In [4]:
BRICS[2]
Out[4]:
'India'
... mas não pode alterá-la!
In [5]:
#BRICS[0] = 40

Você pode aninhar uma tupla -- uma tupla dentro de outra -- porque ela é um objeto Python...
In [6]:
novo_BRICS = (('Brasil', 'Russia', 'India', 'China', 'Africa do Sul'),\
    ('Arabia Saudita', 'Egito', 'Argentina', 'Irã'))
novo_BRICS
Out[6]:
(('Brasil', 'Russia', 'India', 'China', 'Africa do Sul'),
 ('Arabia Saudita', 'Egito', 'Argentina', 'Irã'))
In [7]:
novo_BRICS[0]
Out[7]:
('Brasil', 'Russia', 'India', 'China', 'Africa do Sul')
In [8]:
novo_BRICS[0][0]
Out[8]:
'Brasil'
... bem como é possível concatenar tuplas e formar uma nova tupla...
In [9]:
('Brasil', 'Russia', 'India', 'China', 'Africa do Sul') + \
    ('Arabia Saudita', 'Egito', 'Argentina', 'Irã')
Out[9]:

 'Russia',
 'India',
 'China',
 'Africa do Sul',
 'Arabia Saudita',

 'Argentina',

e para que servem as tuplas?

Dicionário¶

conhecido na literatura como tabela de dispersão, armazena uma coleção de pares (!) chave-valor de tamanho flexível, em que chave e valor são objetos Python -- mas a chave precisa ser imutável!

👉 dica: dicionário provavelmente é a estrutura de dados nativa mais importante do Python, pois é extremamente rápido.

Você pode criar um dicionário...
In [50]:

populacao
Out[50]:

Out[51]:

... você pode acessar um dicionário com a chave...

Conjunto¶

Um conjunto (set) é uma coleção não ordenada de elementos únicos.

👉 dica: podemos pensar neles como dicionários, mas somente com as chaves, sem os valores.

Você pode criar um conjunto...
In [69]:
s1 = set()
s1
Out[69]:
set()
In [70]:
s1 = {'Brasil', 'Russia', 'India', 'China', 'Africa do Sul'}
s1
Out[70]:
{'Africa do Sul', 'Brasil', 'China', 'India', 'Russia'}
em que o seu grande diferencial é o uso de operações específicas da teoria de conjuntos.
In [71]:
a = {'Brasil', 'Russia', 'India', 'China', 'Africa do Sul'}
b = {'Brasil', 'Russia', 'India', 'China', 'Africa do Sul', 'Arabia Saudita', 'Egito', 'Argentina', 'Irã'}
In [72]:
a.union(b)
Out[72]:
{'Africa do Sul',
 'Arabia Saudita',
 'Argentina',
 'Brasil',
 'China',
 'Egito',
 'India',
 'Irã',
 'Russia'}
In [73]:
a.intersection(b)
Out[73]:
{'Africa do Sul', 'Brasil', 'China', 'India', 'Russia'}
In [74]:
a.difference(b)
Out[74]:
set()
In [75]:
b.difference(a)
Out[75]:
{'Arabia Saudita', 'Argentina', 'Egito', 'Irã'}

Quadro-resumo dos métodos de operação em conjuntos do Python¶

FunçãoSintaxe AlternativaDescrição
a.add(x)N/AAdiciona o elemento x ao conjunto a
a.clear()N/AReinicia o conjunto a deixando-o em um estado vazio, descartando todos os seus elementos
a.remove(x)N/ARemove o elemento x do conjunto a
a.pop()N/ARemove um elemento arbitrário do conjunto a, gerando um KeyError se o conjunto estiver vazio
a.union(b)a | bTodos os elementos únicos em a e b
a.update(b)a |= bDefine o conteúdo de a como a união dos elementos em a e b
a.intersection(b)a & bTodos os elementos tanto em a quanto em b
a.intersection_update(b)a &= bDefine o conteúdo de a como a interseccão dos elementos em a e em b
a.difference(b)a - bOs elementos em a que não estão em b
a.difference_update(b)a -= bDefine a com os elementos que estão em a, mas que não estão em b
a.symmetric_difference(b)a ^ bTodos os elementos que estão em a ou em b, mas não em ambos
a.symmetric_difference_update(b)a ^= bDefine a para que contenha os elementos que estão em a ou em b, mas não em ambos
a.issubset(b)N/ATrue se os elementos de a estiverem todos contidos em b
a.issuperset(b)N/ATrue se os elementos de b estiverem todos contidos em a
a.isdisjoint(b)N/ATrue se a e b não tiverem elementos em comum

sorted¶

é comum: percorrer uma coleção de dados com o método sorted, que ordena uma estrutura de dados

👉 dica: o método sorted aceita os mesmos parâmetros do método sort, mas o primeiro gera uma nova coleção, enquanto o segundo opera in-place.

In [87]:

In [88]:

zip¶

é comum: parear elementos de coleções distintas com o método zip

👉 dica: zip pode receber um número arbitrário de coleções, e o número de elementos que ele produz é determinado pela coleção mais curta

In [89]:
ordinal = [1, 2, 3]
portugues = ['um', 'dois', 'tres', 'quatro']
ingles = ['one', 'two', 'three']
frances = ['un', 'deux', 'trois']

for tupla in zip(ordinal, portugues, ingles, frances):
    print(tupla)

(1, 'um', 'one', 'un')
(2, 'dois', 'two', 'deux')
(3, 'tres', 'three', 'trois')

Funções🧑🏽‍🔧 (inclui funções anônimas/lambda 🥷🏽)¶

As funções são o principal e mais importante método de organização de código e reutilização em Python.

Você pode definir sua função:

  • com a palavra reservada def;
  • com argumentos entre parênteses e separados por vírgula;
  • e um comando opcional de retorno com a palavra reservada return

👉 Referência: docs.python.org

In [95]:
def my_function(x, y):
    return x + y
In [96]:
my_function(1, 2)
Out[96]:
3
In [97]:
def function_without_return(x):
    print(x)
In [98]:
function_without_return("hello!")

hello!

Você pode atribuir o valor retornado pela função para uma variável
In [99]:
result = my_function(1, 2)
result
Out[99]:
3
Você pode ter argumentos posicionais (positional arguments) e argumentos nomeados (keyword arguments).
  • Argumentos nomeados: opcionais, que recebem um valor padrão (por atribuição);
  • Argumentos posicionais: obrigatórios, que não recebem um valor padrão; e

🚨 Os argumentos nomeados devem vir depois dos argumentos posicionais, caso existam.

In [100]:
def my_function2(x, y, z=1.5):
    if z > 1:
        return z * (x + y)
    else:
        return z / (x + y)

Out[101]:
0.06363636363636363
In [102]:

Out[102]:
35.49
In [103]:
my_function2(10, 20)
Out[103]:
45.0

Escopo (📭 🆚 📪)¶

Cuidado! As funções podem acessar variáveis em dois escopos diferentes: global e local.

👉 dica: evite usar o escopo global

In [104]:
a = [1, 2, 3]
def func():
    global a
    a.extend([4, 5, 6])
    print(a)
    
func()
print(a)

[1, 2, 3, 4, 5, 6]
[1, 2, 3, 4, 5, 6]
In [105]:
a = [1, 2, 3]
def func():
    a = [1, 2, 3]
    a.extend([4, 5, 6])
    print(a)
    
func()
print(a)

[1, 2, 3, 4, 5, 6]
[1, 2, 3]

Funções Anônimas/Lambda 🥷¶

As funções anônimas são funções em Python que podem ser definidas sem um nome formal.

👉 dica: essa é uma forma de escrever funções, constituídas de uma única instrução, cujo resultado é o valor de retorno.

In [110]:
(lambda x: x*2)
Out[110]:
<function __main__.<lambda>(x)>
In [111]:
(lambda x: x*2)(2)
Out[111]:
4
Para tornar mais claro, perceba que a função anônima acima equivale à função abaixo.
In [112]:
def short_function(x):
    return x * 2
In [113]:
short_function(2)
Out[113]:
4
Note que você pode atribuir a função anônima a uma variável, que assumirá o nome formal da função
In [114]:
short_function = lambda x: x*2
In [115]:
short_function(2)
Out[115]:
4

👉 mais dica: importante para análise de dados porque muitas vezes as funções de transformação de dados aceitarão funções como argumentos

In [116]:
raw_states = [' Ceará ', 'CEará', 'ceará', 'distrito Federal', 
              '    Distrito Federal']

def limpar_estados(raw_states, clean_operation):
    return set([clean_operation(state) for state in raw_states])

def limpar_estado(raw_state):
    return raw_state.lstrip().rstrip().title()

limpar_estados(raw_states, limpar_estado)

{'Ceará', 'Distrito Federal'}

raw_states = [' Ceará ', 'CEará', 'ceará', 'distrito Federal',

limpar_estados = lambda raw_states, clean_operation: set([clean_operation(state) for state in raw_states])

👉 curiosidade: por que lambda é palavra reservada para função anônima?

As funções anônimas têm origem no trabalho de Alonzo Church sobre sua invenção chamada cálculo lambda, onde todas as funções são funções anônimas.

Erros 🚫 e tratamento de exceção ✅¶

Tratar erros ou exceções de modo elegante é uma parte importante na construção de programas robustos.

👉 dica: Em aplicações de análise de dados, muitas funções servem somente para determinados tipos de entrada.

👉 Referência: docs.python.org

Você pode usar a função float para fazer cast de uma string para um número de ponto flutuante...
In [119]:
def converte_para_float(x):
    return float(x)
In [120]:
converte_para_float('3.14')
Out[120]:
3.14
... porém a função falha se os dados de entrada forem string, informando o erro ValueError!
In [121]:
# converte_para_float('frase')

Você pode tratar o erro de forma elegante com as palavras-chave try/except, capturando o erro ValueError...
In [122]:
def converte_para_float(x):
    try:
        return float(x)
    except ValueError:
        return x
In [123]:
converte_para_float('3.14')
Out[123]:
3.14
In [124]:
converte_para_float('frase')
Out[124]:
'frase'
... porém a função falha se os dados de entrada forem tuplas, agora informando o erro TypeError!
In [125]:
# converte_para_float((1,2))

Você pode capturar vários tipos de exceção escrevendo uma tupla de tipos de exceção (com parênteses)
In [126]:
def converte_para_float(x):
    try:
        return float(x)
    except (ValueError, TypeError):
        return x
In [127]:
converte_para_float('3.14')
Out[127]:
3.14
In [128]:
converte_para_float('frase')
Out[128]:
'frase'
In [129]:
converte_para_float((1,2))
Out[129]:
(1, 2)
  • por fim, você pode ter um código que execute somente se o try for bem-sucedido usando else; e
  • você pode ter um código que sempre execute, independentemente de o código no bloco try ter sido bem-sucedido, usando finally.
In [130]:
def converte_para_float(x):
    try:
        y = float(x)
    except (ValueError, TypeError):
        print('deu erro')
        return x
    else:
        print('funcionou')
        return y
    finally:
        print('vou sempre executar, independentemente do resultado')
In [131]:
converte_para_float('3.14')

funcionou
vou sempre executar, independentemente do resultado
Out[131]:
3.14
In [132]:
converte_para_float('frase')

deu erro
vou sempre executar, independentemente do resultado
Out[132]:
'frase'
In [133]:
converte_para_float((1,2))

deu erro
vou sempre executar, independentemente do resultado
Out[133]:
(1, 2)
